# Task: Real LLM Provider Readiness and Diagnostics

## Description

Make real OpenAI-compatible provider mode operational and diagnosable without exposing secrets. Preserve mock mode for tests and offline demo, but ensure real provider failures are clear and recoverable.

## Acceptance Criteria

- [ ] `.env.example` documents `LLM_PROVIDER`, `LLM_API_KEY`, `LLM_BASE_URL`, `LLM_MODEL`, and timeout settings.
- [ ] Backend maps missing API key, timeout, HTTP error, invalid provider, and invalid model response into clear Agent errors.
- [ ] A provider diagnostic endpoint or service can verify model connectivity with a safe dry-run.
- [ ] API key values are never returned to frontend responses, logs, or timeline snapshots.
- [ ] Existing mock-mode tests still pass.

## Technical Details

- Work in `backend/app/agent/llm_client.py`, `backend/app/core/config.py`, `backend/app/api`, `backend/app/schemas`, and tests.
- Use OpenAI-compatible `/chat/completions` first; do not add SDK dependency unless it removes real complexity.
- Do not create or edit `.env`.

## Dependencies

None.
