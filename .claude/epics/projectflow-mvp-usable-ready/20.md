# Task: Assignment, Push, Risk, and Replan Usability Pass

## Description

Bring the execution-loop Agent outputs up to usable quality. Assignment recommendations, active push cards, risk analysis, and replan proposals must be specific enough for a student team to act on.

## Acceptance Criteria

- [ ] Assignment reasons cite member skill, availability, preference, or constraint.
- [ ] Active push cards include goal, start suggestion, completion standard, due date, and reason.
- [ ] Risk cards include evidence from task status, check-in, workload, dependency, deadline, or review pressure.
- [ ] Replan proposals include before/after, impact, reason, and explicit confirmation requirement.
- [ ] High-risk suggestions do not directly modify finalized task ownership.
- [ ] Tests cover one blocker + availability-drop scenario using real schema-shaped Agent output.

## Technical Details

- Work in Agent modules, schemas, services, project dashboard components, and tests as needed.
- Preserve deterministic fallback, but make fallback clearly less authoritative than real success.
