# Task: Agent Infrastructure and Structured Outputs

## Description

Create the single Coordinator Agent foundation, LLM provider adapter, prompt boundaries, output schemas, validation, fallback pipeline, and timeline logging required by all agent modules.

## Acceptance Criteria

- [x] `backend/app/agent` contains coordinator, workflow, prompts, LLM client, output schemas, and module structure.
- [x] LLM provider configuration supports at least a mock provider and an OpenAI-compatible provider through environment variables.
- [x] Agent output schemas exist for direction card, stage plan, task breakdown, assignment recommendation, assignment negotiation, active push, check-in analysis, risk analysis, and replan.
- [x] Output validation uses Pydantic and fails closed.
- [x] Fallback pipeline supports JSON repair, one retry, template fallback, and timeline event logging.
- [x] AgentEvent records include input snapshot, output snapshot, event type, status, and reasoning summary where available.
- [x] Tests cover schema validation and fallback behavior without requiring a real API key.

## Technical Details

- The Coordinator reads WorkspaceState and delegates generation to module functions.
- Agent modules must not write directly to the database.
- High-risk suggestions must set `requires_confirmation`.
- Prompts should explicitly prevent fabricated members, tasks, stages, or assignments.
