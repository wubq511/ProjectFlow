# #19 设计：Polish Project Intake, Workspace, and Member Profile UX

## 目标

将项目录入、工作区管理、成员资料三个核心模块的 UX 打磨至接近产品级，消除所有已知的粗糙感。目标是在 2026-05-31 前完成，作为 MVP 演示前的最终 UX 加固。

## 范围

仅修改前端组件和样式，不改动后端 API、数据库模型或 Agent 逻辑。

## 已识别的 UX 问题

### 1. Project Intake Form
- 字段布局简陋，所有输入框垂直堆叠无分组
- 缺少字段说明和示例占位符
- 无实时校验反馈（如截止日期必须在未来）
- 提交后无成功状态，直接跳转
- 资源输入面板与主表单视觉不连贯

### 2. Workspace Management
- 创建工作区表单过于简单，缺少团队规模、项目类型等上下文
- 工作区列表页无空状态设计
- 邀请成员面板无复制链接反馈
- 无工作区设置/编辑功能

### 3. Member Profile
- 资料填写步骤过长，无进度保存
- 技能标签输入体验差（无自动完成、无热门标签建议）
- 可用时间选择器不直观
- 无资料完成度提示

## 设计方案

### 总体原则
- **渐进式展示**：先给核心，细节按需展开
- **即时反馈**：每个操作都有视觉或文本反馈
- **防错设计**：在错误发生前引导，而非事后报错
- **移动端优先**：所有表单在 375px 宽度下可用

### Project Intake 改进

#### 布局重构
```
[项目基本信息]
  项目名称*    截止日期*
  项目描述*    [多行文本]
  
[项目详情]
  项目类型 [单选卡片：课程作业/比赛/创业/研究]
  团队规模 [数字步进器]
  预期交付物 [标签输入]
  
[资源与约束]
  [展开/折叠] 现有资源
  [展开/折叠] 已知限制
  
[提交]
  [保存草稿] [开始规划 →]
```

#### 具体改动
1. **分块卡片式布局**：使用 `Card` 组件将表单分为「基本信息」「项目详情」「资源与约束」三块
2. **项目类型选择器**：用图标+文字的卡片网格替代下拉选择，4 个选项：课程作业、比赛、创业、研究
3. **交付物标签输入**：类似邮箱收件人输入，支持逗号/回车分隔，可删除标签
4. **实时校验**：
   - 项目名称：非空，2-50 字符
   - 截止日期：必须在未来 7 天内
   - 描述：非空，10-500 字符
   - 校验错误即时显示在字段下方，使用 `text-destructive` 颜色
5. **草稿保存**：localStorage 自动保存，页面刷新后恢复
6. **提交反馈**：提交按钮显示 loading spinner，成功后显示 toast「项目创建成功，正在生成规划建议...」
7. **资源面板整合**：将 ResourceInputPanel 的展开/折叠状态与主表单统一，使用一致的卡片样式

### Workspace Management 改进

#### 创建工作区
```
[工作区信息]
  工作区名称*
  工作区描述
  
[团队上下文]
  团队规模 [1-2/3-5/6-10/10+]
  主要场景 [课程/比赛/创业/其他]
  
[高级设置]
  [展开] 工作区标识色 [颜色选择器]
```

#### 工作区列表
- 空状态：插图 + 「创建你的工作区」CTA
- 卡片式列表：显示名称、成员数、项目数、最后活跃时间
- 悬停显示「进入」「设置」操作

#### 邀请成员
- 复制链接后显示「已复制」tooltip 2 秒
- 支持直接输入邮箱发送邀请（模拟）

### Member Profile 改进

#### 步骤简化
将原来的单页长表单改为 3 步向导：
1. **基本信息**：姓名、角色、专业年级
2. **技能与经验**：技能标签（带热门建议）、过往项目
3. **可用时间**：周可用小时数、偏好工作时段

#### 技能标签输入
- 输入时显示热门技能建议（前端写死 20 个常见技能）
- 支持点击建议快速添加
- 已选标签显示为可删除的 badge
- 最多 10 个标签

#### 完成度提示
- 顶部进度条显示资料完成度
- 完成度 < 80% 时提示「完善资料可获得更精准的分工建议」

## 样式规范

### 表单字段
- 标签：`text-sm font-medium mb-1.5`
- 输入框：`h-10 rounded-md border-input`
- 错误提示：`text-xs text-destructive mt-1`
- 字段间距：`mb-5`

### 卡片区块
- 背景：`bg-card`
- 圆角：`rounded-xl`
- 内边距：`p-6`
- 阴影：`shadow-sm`
- 标题：`text-base font-semibold mb-4`

### 按钮层级
- Primary：`bg-primary text-primary-foreground`
- Secondary：`bg-secondary text-secondary-foreground`
- Outline：`border bg-background`
- Ghost：`hover:bg-accent`

### 动画
- 卡片展开：`animate-in fade-in slide-in-from-top-2 duration-200`
- 错误抖动：`animate-[shake_0.3s_ease-in-out]`
- 按钮 loading：`opacity-50 cursor-wait`

## 依赖

无需新增依赖，全部使用现有技术栈：
- shadcn/ui 组件（Card, Button, Input, Label, Badge, Progress, Tooltip, Dialog, Sheet）
- Tailwind CSS 动画
- Framer Motion（用于复杂过渡，可选）
- lucide-react 图标

## 风险与回滚

- **风险**：改动面较广，可能引入回归问题
- **缓解**：每个模块独立修改，改完立即在浏览器验证
- **回滚**：所有改动纯前端，git revert 即可

## 数据模型对齐说明

### Project Intake
- 新增字段 `project_type`（字符串，可选）："coursework" | "competition" | "startup" | "research"
- 新增字段 `team_size`（整数，可选）：团队规模
- 新增字段 `deliverables`（字符串数组，可选）：预期交付物列表
- 这些字段仅用于前端展示和 localStorage 草稿，**不修改后端 ProjectCreate schema**，在提交时丢弃或存入 localStorage 草稿

### Workspace
- 新增字段 `team_size_range`（字符串，可选）："1-2" | "3-5" | "6-10" | "10+"
- 新增字段 `primary_use_case`（字符串，可选）："course" | "competition" | "startup" | "other"
- 这些字段仅用于前端展示，**不修改后端 WorkspaceCreate schema**

### Member Profile
- 保持现有 schema 不变，完成度计算纯前端逻辑
- 技能标签仍用逗号分隔字符串存储，与后端兼容
